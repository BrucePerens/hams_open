Database Management System (DBA Toolkit)

The Database Management module provides a powerful set of tools designed for Site Reliability Engineers (SREs) and Database Administrators (DBAs) to monitor and optimize the Odoo PostgreSQL backend directly from the web interface.

Core Features

  • Table Health & Bloat Monitoring: Track real-time statistics on table size, live vs. dead tuples, and cache hit ratios.
  • Index Usage Analysis: Identify unused or oversized indexes that may be slowing down write operations.
  • Slow Query Tracking (APM): View the most time-consuming SQL queries to pinpoint performance bottlenecks in application code.
  • Active Session Management: Monitor currently running database backends and safely terminate runaway or locked queries.
  • PostgreSQL Tuning Wizard: Automatically calculate and apply optimal database settings based on your server's RAM and CPU resources.
  • High Availability Orchestration: Generate production-ready configuration templates for Patroni and PgBouncer.

How to Use Table Health & Bloat Tools

Navigating to Database Health > Table Health & Bloat allows you to see which tables are consuming excessive disk space due to "dead tuples" (records that have been deleted or updated but not yet physically removed).

  1. Identify tables with a high Dead % (typically over 20%).
  2. Select the bloated tables using the checkboxes in the list view.
  3. Click the Vacuum Analyze Selected button to trigger a manual cleanup. This process runs in the background and updates PostgreSQL statistics.

Note: This module automatically integrates with PagerDuty to alert administrators if critical tables exceed safe bloat thresholds.

Terminating Runaway Sessions

If the system becomes unresponsive due to a locked process, go to Database Health > Active Sessions.

  1. Review the Duration and Query columns to find the offending process.
  2. Select the session(s) you wish to stop.
  3. Click Kill Selected Queries. You will be asked for confirmation before the pg_terminate_backend signal is sent.

Optimizing Performance

The Optimization Wizard (under Configuration & Tuning) simplifies PostgreSQL tuning:

  1. Enter your server's total RAM (in GB) and CPU core count.
  2. Select your storage type (SSD is recommended for Odoo).
  3. Click Calculate & Apply Optimizations.

The system will use industry-standard DBA algorithms to write new settings (like shared_buffers and work_mem) to your postgresql.auto.conf file and reload the configuration.

Warning: Major memory changes require a manual restart of the PostgreSQL service to take full effect.

Security and Access Control

Because these tools interact directly with the database engine, access is restricted to System Administrators. All operations are performed using a dedicated micro-privilege service account to ensure the highest level of security and auditability.
