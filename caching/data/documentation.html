1. Overview

The Caching module dramatically speeds up your website by saving essential design files (like JavaScript, CSS, and UI icons) directly in your browser's local storage. This means that after the first time you visit, the website loads almost instantly because it doesn't have to wait for the same files to download over the internet again. This module acts like a high-speed personal library for your web browser, keeping the website's interface ready for use at a moment's notice.

2. How it Works

When you visit the site, a "Service Worker" is installed in your browser. This is a small, specialized script that acts as an intelligent assistant for your web requests.

  • Smart Asset Interception: It automatically identifies and saves Odoo's core assets and custom module files.
  • Local Fulfillment: On subsequent visits, these files are delivered from your computer's storage with near-zero latency, even on slow connections.
  • Automatic Background Updates: If the website design is updated, your browser will automatically detect the change on the next visit, download the new versions in the background, and seamlessly update your local cache.

3. Important Safety Features

  • Intelligent Quota Management: Browsers limit how much data a Service Worker can save. To prevent the browser from "panicking" and deleting the entire cache, our assistant calculates exactly how much space is safe. It prioritizes small, essential UI files and automatically skips very large media files.
  • Privacy-First Architecture: The module only saves public technical assets required to display the website. It never saves your personal information, login credentials, or private data.
  • Real-time Compatibility: Live communication features like chat and notifications are explicitly bypassed by the caching system to ensure they always provide real-time updates.

4. Administrator Controls

Administrators can fine-tune the caching behavior from the Website Settings (Configuration > Settings > Website section):

  • Safe Quota (MB): Adjust the maximum total size of files the browser is allowed to save. The default is 35MB, which is safe for most modern browsers. If your website's assets exceed this limit, the module will automatically prioritize essential files and skip the heaviest ones to maintain stability.
  • Force Cache Invalidation: If you have made major visual changes (like updating logos, brand colors, or custom scripts) and want to ensure all users see the update immediately on their next visit, click the Invalidate Cache Now button. This increments the cache version and forces all browsers to purge their saved assets and download fresh ones.

5. Multi-Website Management

Each website in your Odoo system has its own independent cache settings. When you change the Safe Quota or click Invalidate Cache Now, it only affects the website you are currently configuring. This allows you to manage multiple brands and websites on the same server without them interfering with each other's performance optimization.

6. Troubleshooting

If the website layout appears inconsistent or "broken" after a major update, it is possible that some browsers are still holding onto stale cached files. An administrator can resolve this globally by using the Invalidate Cache Now button in the Website Settings. For individual users, a simple page refresh is usually enough to trigger the Service Worker's background update mechanism.

7. Developer Tips

For developers creating new modules:

  • Assets in /your_module/static/src/js/ and /your_module/static/src/css/ are automatically cached.
  • To force a cache refresh during development, you can either restart the Odoo server or use the Invalidate Cache Now button in settings.
  • Avoid putting very large files (over 5MB) in the static/ folder, as they might be excluded from the cache to preserve the storage quota.
