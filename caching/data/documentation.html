1. Overview

The Caching module speeds up your website by saving important files directly in your browser. This means that after the first time you visit, the website loads almost instantly because it doesn't have to download the same files again. This module acts like a high-speed personal library for your web browser, keeping the most important parts of the website ready for use at a moment's notice.

2. How it works

When you visit the site, a "Service Worker" is installed in your browser. This is like a small assistant that remembers the website's design (its "assets").

  • Smart Saving: It automatically saves JavaScript, CSS, and images used by the website.
  • Fast Loading: Next time you visit, these files are loaded from your computer's storage instead of over the internet.
  • Automatic Updates: If the website owner updates the design, your browser will automatically notice and update its saved files.

3. Important Safety Features

  • Storage Limit: Browsers only allow a certain amount of space for saved files. Our assistant calculates exactly how much space is safe and only saves the most important files. It skips very large files to prevent filling up your storage.
  • Privacy & Security: It only saves public files needed to display the website. It never saves your personal information, login details, or private data.
  • Real-time Communication: Chat and other live features are not affected and will continue to work perfectly.

4. Administrator Controls

Administrators can manage the Caching Service Worker from the Website Settings (Configuration > Settings > Website section):

  • Safe Quota (MB): Adjust the maximum total size of files the browser is allowed to save. If the website's assets exceed this, the module will automatically prioritize small, essential files and skip large ones. This prevents the website from taking up too much space on a visitor's device.
  • Cache Invalidation: If you have made changes to the website and want to ensure all users see the update immediately, click the Invalidate Cache Now button. This increments the cache version and forces all browsers to clear their saved assets on their next visit. This is useful after major updates to logos, colors, or website behavior.

5. Troubleshooting

If the website looks strange after an update, it's possible that some old files are still being used. Although the module handles this automatically, an administrator can use the "Invalidate Cache Now" button described above to fix the issue globally.
