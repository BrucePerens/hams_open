☁️ Cloudflare Edge Orchestration (cloudflare)

The Cloudflare Edge Orchestration module acts as a command center for your Cloudflare CDN and Web Application Firewall (WAF). It automates caching, security, and connectivity between your Odoo server and Cloudflare's global network, offloading significant processing from your Odoo workers to Cloudflare's edge.

1. Configuration

To start using this module, you need to provide your Cloudflare credentials. Navigate to Settings > Website > Cloudflare Edge.

  • CF API Token: A token created in your Cloudflare dashboard. It requires permissions for:
    • Zone.Cache Purge (to clear cache)
    • Zone.Firewall Services (to manage WAF and IP bans)
    • Account.Cloudflare Tunnel (optional, if using Zero Trust Tunnels)
  • CF Zone ID: The unique identifier for your domain in Cloudflare, found on the domain's overview page.
  • CF Account ID: Your Cloudflare account ID, required for provisioning Zero Trust Tunnels.
  • Turnstile Secret: The secret key for Cloudflare Turnstile, used for backend verification of invisible CAPTCHAs.

2. Cache Management

This module ensures your website is blazing fast by aggressively caching content while providing smart invalidation.

  • Automatic Caching: Static assets (images, CSS, JS) are instructed to be cached for 1 year. Dynamic website pages are cached for 24 hours at the edge for public users.
  • Proactive Invalidation: When a website page, blog post, or product is edited, Odoo automatically tells Cloudflare to purge that specific URL from the global cache.
  • Static Asset Sync: On server boot, the module scans your Odoo installation for changed static files. If any are detected, it triggers a global purge for the odoo-static-assets tag.
  • Manual Purge Wizard: Go to Website > Configuration > Cloudflare > Purge Cache to manually clear the cache.
    • Purge Everything: Clears the entire cache for the zone.
    • Purge Specific URLs: Provide relative paths (e.g., /shop) or full URLs.
    • Purge Specific Cache-Tags: Use relational tags to purge groups of content.

3. Security & Web Application Firewall (WAF)

Leverage Cloudflare's world-class security to protect your Odoo instance.

  • Automated IP Banning: Integrated with honeypots, this module can instantly ban malicious IPs at the Cloudflare edge. View and manage these under Website > Configuration > Cloudflare > IP Bans.
  • WAF Rule Management: Manage Cloudflare Firewall rules directly from Odoo.
    • Pull: Import your existing Cloudflare rules into Odoo for management.
    • Push: Deploy your Odoo-defined rules to Cloudflare.
    Go to Website > Configuration > Cloudflare > WAF Rules to manage these.
  • Zone Settings Wizard: Quickly adjust edge settings like Security Level (e.g., "Under Attack" mode), Development Mode, and Browser Cache TTL.

4. Cloudflare Zero Trust Tunnels

Securely expose your Odoo instance to the internet without opening any inbound ports on your firewall.

  • Provisioning: In Cloudflare settings, click Generate Tunnel Command. The module creates the tunnel via API and provides the exact cloudflared command to run on your host.
  • Management: Track active tunnels under Website > Configuration > Cloudflare > Tunnels. Use Sync Tunnels to refresh the list from your Cloudflare account.

5. Integration for Developers

The module provides a robust API for other modules to leverage Cloudflare capabilities:

  • IP Banning: env['cloudflare.waf'].ban_ip(ip_address, notes="Reason")
  • Cache Purging: env['cloudflare.purge.queue'].enqueue_urls(['/path']) or enqueue_tags(['tag'])
  • CAPTCHA Verification: env['cloudflare.turnstile'].verify_token(token)
  • Edge Context: env['cloudflare.utils'].get_request_context() returns geo-location and threat data from Cloudflare headers.
