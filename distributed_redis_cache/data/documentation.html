Distributed Redis Cache

This module helps keep your Odoo cluster fast and consistent by using Redis to share cached data between different Odoo servers. It ensures that when data changes on one server, all other servers are immediately notified to update their own caches.

How it Works

Normally, Odoo keeps its own local cache on each server. If you have many servers, they can sometimes get out of sync. This module solves that by:

  1. Sending a signal through the database when data changes.
  2. A background helper (the "Cache Manager") picks up this signal and tells Redis about it.
  3. Redis broadcasts the message to every Odoo server in your cluster.
  4. Each Odoo server receives the message and clears the specific part of its memory that was affected, ensuring the next time it needs that data, it gets the fresh version.

Using the Management UI

Administrators can manage the cache through the Distributed Cache menu.

  • Checking Status: Click the "Check Redis Status" button to see if Odoo can successfully talk to the Redis server. If it's working, you'll see a green success message. If Redis is down, Odoo will automatically switch to its safe "Fail-Open" mode and continue working using its own local memory.
  • Manual Invalidation: If you ever suspect that the cache is out of date, you can manually clear it for a specific type of data (a "Model"). Select the model (for example, "User") from the dropdown list and click "Invalidate Model Cache". This will clear that data from every server in your cluster immediately. Security and Performance

  • Zero-Sudo: The module is built to be secure and never uses "Superuser" powers unnecessarily. It uses a dedicated background account for its operations. [@ANCHOR: story_zero_sudo_cache_manager]
  • Smart Clearing: Unlike standard Odoo which sometimes clears everything at once, this module only clears the specific data that changed. This keeps your system running fast even when many changes are happening.
  • Fail-Open: If the Redis server has a problem, your Odoo site won't crash. It will just behave like a standard Odoo setup until Redis comes back online.
