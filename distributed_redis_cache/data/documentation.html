Distributed Redis Cache

The Distributed Redis Cache module is a performance tool designed for Odoo environments with multiple servers. It ensures that when data changes on one server, every other server in your cluster is updated instantly, preventing "stale data" errors.

Overview

Normally, Odoo servers work independently. If you update a price or a user's permissions on Server A, Server B might not realize it for some time. This module links them all to a central "brain" (Redis) so they can share information about what has changed in real-time.

How it Works

  1. Detection: Odoo automatically identifies when you save a record in a model (like a User, Product, or Customer).
  2. Broadcast: A background service (the Cache Manager) immediately relays this change to the shared memory server (Redis).
  3. Refresh: All other Odoo servers in your cluster are listening to Redis. As soon as they hear the signal, they instantly clear their local memory for that specific data type and fetch the latest version from the database.
Smart Cleanup: Instead of clearing everything and slowing down your system, this module only clears the specific data that actually changed.

User Instructions

Managing the Cache

The Distributed Cache Manager is your central dashboard for ensuring synchronization across your Odoo cluster. Administrators can find these tools under the Distributed Cache menu in Odoo Settings.

Checking System Health

Click Check Redis Status to see if Odoo is talking to the shared memory server.

  • Green Success: Everything is working perfectly.
  • Red Error: The shared memory is unreachable. Don't worry—Odoo will automatically switch to "Fail-Open" mode and use standard local memory so your website stays online.

Configuration

Administrators can configure the Redis connection under the Settings menu in the Distributed Redis Cache section.

  • Redis Host: The hostname or IP address of your Redis server.
  • Redis Port: The port Redis is listening on (default is 6379).
  • Redis Password: The password required to connect to your Redis instance.

Manual Refresh

If you suspect a server is showing old information, you can manually trigger a refresh:

  1. Select the Data Type (called a "Model", like User or Partner) from the list.
  2. Click Invalidate Model Cache.
  3. All servers in your cluster will instantly refresh their memory for that specific data type.

Privacy and Isolation

This module is built for security. It automatically keeps data from different websites or different companies separate. A user on Website A will never see cached data intended for a user on Website B.

Technical Benefits

  • Secure: Uses "Zero-Sudo" architecture—background tasks have only the minimum access they need.
  • Reliable: Designed to never crash your site even if the cache server goes down.
  • Efficient: Uses optimized commands to handle millions of records without lagging.
Developer Tip: Use the @distributed_cache() decorator on your custom model methods to automatically benefit from cluster-wide caching and multi-tenant isolation.

Troubleshooting

If updates aren't appearing instantly:

  • Check Status: Use the "Check Redis Status" button.
  • Check the Service: Ensure the "Cache Manager" background service is active on your host.
  • Be Patient: While usually instant, network signals can sometimes take a fraction of a second to travel between servers.
