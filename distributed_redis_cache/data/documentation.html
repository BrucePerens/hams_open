Distributed Redis Cache

This module helps multiple Odoo servers stay in sync by sharing a central cache. It ensures that when data changes on one server, the other servers are immediately notified to update their local information.

Overview

Normally, each Odoo server keeps its own copy of frequently used data (like user settings or configuration) in its own memory. If you have several servers running at once, they might end up with different versions of the same data. This module uses a central Redis server to keep everyone on the same page.

How It Works

The system follows these steps to keep data consistent:

  1. When data in a model (like a User or Partner) is changed on one Odoo server, it sends a notification.
  2. A background service (the Cache Manager) picks up this notification and broadcasts it to all other Odoo servers.
  3. Every server receives this message and clears its local memory for that specific model, ensuring the next time the data is needed, it is fetched fresh.

Zero-Sudo Architecture

Distributed Redis Cache strictly adheres to the Zero-Sudo principle. No .sudo() calls are used in the core caching logic or controllers. All background tasks and administrative operations are performed using dedicated service accounts with minimal privileges.

  • Service Account: cache_manager_sys is used by the standalone daemon to connect to PostgreSQL and listen for invalidation events. [@ANCHOR: story_zero_sudo_cache_manager]
  • Micro-Privilege: UI access is controlled by the Cache Manager group, which is granted to specific administrators without requiring full Odoo Settings access.

Configuration

The following environment variables can be used to configure the Redis connection:

  • REDIS_HOST: Redis server hostname (default: redis or 127.0.0.1).
  • REDIS_PORT: Redis server port (default: 6379).
  • REDIS_PASSWORD: Redis server password (optional).

For the daemon, these can also be placed in /var/lib/odoo/daemon_keys/cache_manager.env.

Usage

Decorator

Use the @distributed_cache() decorator on api.model functions to automatically generate cache keys and write them to Redis with a 24h TTL. 

from odoo.addons.distributed_redis_cache.redis_cache import distributed_cache

class MyModel(models.Model):
    _name = 'my.model'

    @distributed_cache()
    def my_cached_method(self, arg1):
        # ... logic ...
        return result

Manual Invalidation

Use invalidate_model_cache(env, model_name) to forcibly flush local WSGI memory. To trigger a cross-worker alert, use notify_model_invalidation(env, model_name). [@ANCHOR: notify_model_invalidation_logic]

Configuration UI

The configuration UI allows for manual cache management and status monitoring. [@ANCHOR: manual_cache_invalidation]

You can check if the Redis connection is healthy. [@ANCHOR: check_redis_status_logic]

Management UI

The module provides a dedicated UI to manage the cache and check Redis status. [@ANCHOR: distributed_cache_view]

Fail-Open Resilience

If Redis is unavailable, the system automatically falls back to a local memory cache (_local_cache) to ensure continuous operation, albeit without cross-worker phase coherence.
