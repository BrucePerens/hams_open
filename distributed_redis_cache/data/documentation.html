Distributed Redis Cache

Fine-grained distributed caching and phase coherence for horizontally scaled Odoo clusters.

Overview

Standard Odoo @tools.ormcache relies on a local worker registry cache, which can drift out of sync in horizontally scaled, multi-node environments. This module provides a fine-grained, distributed Redis-backed cache that enforces strict phase coherence across the entire cluster.

Architecture

The invalidation pipeline works as follows:

  1. An Odoo worker mutates a cached model and fires a PostgreSQL NOTIFY on the distributed_cache_invalidation channel.
  2. The standalone cache_manager.py daemon receives the NOTIFY and publishes the payload to the Redis odoo_cache_invalidation_bus pub/sub channel.
  3. A background thread running inside every Odoo WSGI worker's ir.http middleware intercepts the Redis broadcast and instantly queues the model for local cache flushing before serving its next HTTP request.

Zero-Sudo Architecture

Distributed Redis Cache strictly adheres to the Zero-Sudo principle. No .sudo() calls are used in the core caching logic or controllers. All background tasks and administrative operations are performed using dedicated service accounts with minimal privileges.

  • Service Account: cache_manager_sys is used by the standalone daemon to connect to PostgreSQL and listen for invalidation events.
  • Micro-Privilege: UI access is controlled by the Cache Manager group, which is granted to specific administrators without requiring full Odoo Settings access.

Configuration

The following environment variables can be used to configure the Redis connection:

  • REDIS_HOST: Redis server hostname (default: redis or 127.0.0.1).
  • REDIS_PORT: Redis server port (default: 6379).
  • REDIS_PASSWORD: Redis server password (optional).

For the daemon, these can also be placed in /var/lib/odoo/daemon_keys/cache_manager.env.

Usage

Decorator

Use the @distributed_cache() decorator on api.model functions to automatically generate cache keys and write them to Redis with a 24h TTL. 

from odoo.addons.distributed_redis_cache.redis_cache import distributed_cache

class MyModel(models.Model):
    _name = 'my.model'

    @distributed_cache()
    def my_cached_method(self, arg1):
        # ... logic ...
        return result

Manual Invalidation

Use invalidate_model_cache(env, model_name) to forcibly flush local WSGI memory. To trigger a cross-worker alert, use notify_model_invalidation(env, model_name).

Management UI

The module provides a dedicated UI to manage the cache and check Redis status.

Fail-Open Resilience

If Redis is unavailable, the system automatically falls back to a local memory cache (_local_cache) to ensure continuous operation, albeit without cross-worker phase coherence.
