Hams Helpdesk

Hams Helpdesk provides Zero-Sudo compliant ticketing and issue tracking. It deeply integrates with the PagerDuty module via an agnostic Adapter API, allowing automated ticket creation and calendar event scheduling during system incidents.

Zero-Sudo Architecture

This module strictly adheres to the Zero-Sudo mandate. Background operations, automated routing, and notifications are performed using dedicated service accounts:

  • Helpdesk Service Account: (hams_helpdesk.user_helpdesk_service) Handles ticket assignments, notifications, and handoff execution.
  • PagerDuty Service Account: (pager_duty.user_pager_service_internal) Used to safely query on-duty status from the calendar.

Ticket Lifecycle

Tickets progress through stages: New, In Progress, Resolved, and Closed. Transitioning a ticket automatically notifies the reporting customer.

Formal Shift Handoff

Continuity is maintained via a formal handoff wizard. Operators are required to provide detailed briefing notes when transferring ownership, ensuring the next shift has all the necessary context to proceed with resolution.

Multi-Website Awareness & Security

The module is fully compatible with multi-website Odoo instances. Tickets are associated with the website where they were created. Portal users will only see tickets related to the website they are currently browsing, ensuring strict data segregation between different web portals.

Access is strictly controlled via micro-privileges. Portal users can only view and message their own tickets and are prohibited from modifying administrative fields such as Status, Assignee, or Priority.

Multi-Company Data Isolation

Hams Helpdesk provides enterprise-grade data isolation. All tickets are tied to a company_id. Standard operators and managers can only see tickets belonging to companies they have access to. Similarly, portal users are strictly restricted to tickets belonging to their authorized companies, ensuring that sensitive support information is never leaked across business boundaries.

Portal Access for Customers

Customers can access their tickets via the web portal. After logging in, they can see a list of their tickets, view the status of each ticket, and communicate with the helpdesk team by posting messages in the chatter. The ticket list is filtered based on the portal website to ensure privacy.

How to use the Helpdesk

Creating a Ticket

Tickets can be created from the Helpdesk menu in the backend. When a ticket is created, it is automatically assigned to the on-duty administrator if PagerDuty integration is active. The reporting customer is automatically subscribed to receive updates. If a ticket is created via a website portal, it is automatically tagged with that website.

Managing a Ticket

Operators can move tickets through different stages (New, In Progress, Resolved, Closed). Each stage change triggers an automated email notification to the customer, keeping them informed about the progress. Critical incidents should be prioritized and moved to "In Progress" as soon as investigation begins.

Executing a Handoff

To hand off a ticket to another operator, use the "Shift Handoff" button. This will open a wizard where you can select the next assignee and enter mandatory handoff notes. This ensures that no context is lost during shift changes. The handoff history is recorded in the ticket's chatter for audit purposes.
