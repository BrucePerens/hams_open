Binary Downloader Facility

The Binary Downloader is a highly secure, database-backed orchestration module designed to provide static executable dependencies (such as kopia, etcd, and cloudflared) to other Odoo subsystems at runtime without exposing the underlying OS to Arbitrary File Write vulnerabilities.

1. The Security Problem

Traditional dependency provisioners rely on flat JSON or YAML files stored on disk. If an attacker discovers a Directory Traversal or File Write vulnerability in another application, they could subtly alter the flat file to point the download URL to a malicious payload. When Odoo attempts to download the dependency, it installs and executes the malware as the Odoo service user.

2. The DB-Backed Solution

This module eliminates flat-file manifests entirely. Download targets, cryptographic SHA-256 checksums, and extraction rules are stored securely within PostgreSQL via the binary.manifest model.

  • Strict Access Control: Only users belonging to the base.group_system (Administrators) can modify the manifest URLs or checksums.
  • Cryptographic Enforcement: Before a downloaded binary is moved into the execution path (hams_bin), its SHA-256 hash is mathematically verified against the database record. Mismatches trigger an instant deletion and an abort.
  • Permission Locking: Extracted binaries are strictly locked to 0o750 (rwxr-x---) to prevent horizontal privilege escalation from other OS users.

3. Usage

Other modules can safely request a dependency by invoking the central method: 

bin_path = self.env["binary.manifest"].ensure_executable("kopia")

The module uses the user_binary_downloader_service service account for all resolution and download operations, ensuring strict Zero-Sudo compliance.
