Binary Downloader Facility

The Binary Downloader is a secure system that provides technical dependencies (helper programs) like kopia (for backups), etcd (for distributed configuration), or cloudflared (for secure tunnels) to Odoo automatically.

1. Purpose and Automation

Modern Odoo deployments often rely on external high-performance tools written in languages like Go or Rust. Instead of requiring manual system administration to install these tools on the host server, the Binary Downloader automates the lifecycle of these dependencies.

When an Odoo module needs a specific program, it asks the Binary Downloader to "ensure" it is available. If the program is missing, it is downloaded, verified for integrity, and installed into a protected internal directory.

2. Operating the Module

While most operations happen automatically, administrators can monitor and trigger installations manually:

  1. Navigate to Settings -> Technical -> Binary Manifests.
  2. Review the list of binaries. The Installed column indicates if the dependency is currently available on the system PATH or in Odoo's private binary directory (hams_bin).
  3. To manually install or update a dependency, click the Install button on the list view, or open the record and click Install Now.
  4. If an installation fails, a detailed notification will explain the cause (e.g., network error or checksum mismatch).

3. Security Architecture

Security is at the core of the Binary Downloader design:

  • Cryptographic Verification: Every downloaded file is verified against a SHA-256 checksum. If the file's digital fingerprint doesn't match the expected value, the installation is aborted to protect against tampering.
  • Advisory Locking: The system uses database-level locks to ensure that multiple Odoo workers don't try to install or update the same binary at the same time, preventing file corruption.
  • Micro-Privilege: Downloads and installations are performed using a restricted service account (binary_downloader_service) to minimize system risk.
  • Path Sanitization: Strict protections are in place to prevent "Tar Slip" vulnerabilities and unauthorized symbolic links when extracting archives.
  • Isolation: Binaries are stored in a dedicated hams_bin folder within Odoo's data directory, kept separate from system-wide binaries.

4. Troubleshooting Common Issues

  • Checksum Failures: If you see a "Security Alert: Checksum mismatch", it means the file downloaded from the internet does not match the fingerprint in Odoo's database. This could be due to a corrupted download or a change in the remote file. Contact your technical support.
  • System Architecture: Automatic installation is currently optimized for 64-bit Linux (x86_64) systems. Other architectures may require manual installation via the system package manager.
  • Network Connectivity: The Odoo server must have HTTPS access to the download URLs (e.g., github.com) to fetch the dependencies.
