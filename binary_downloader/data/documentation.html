Binary Downloader Facility

The Binary Downloader is a secure system that provides technical dependencies (like kopia for backups or cloudflared for secure connections) to Odoo automatically. You won't usually need to interact with it directly, as other modules handle it behind the scenes.

1. What it Does

Some Odoo features require special helper programs (binaries) to work. Instead of requiring you to manually install these on your server, the Binary Downloader can fetch them automatically from trusted sources, verify they haven't been tampered with, and make them available for Odoo to use.

2. How to Use It

If a module requires a binary that isn't installed yet, you can manage it through the Binary Manifests menu:

  1. Navigate to Settings -> Administration -> Binary Manifests.
  2. You will see a list of required binaries (like kopia or cloudflared) and whether they are currently "Installed".
  3. If a binary is not installed, you can click the Install button. Odoo will then download and set it up for you.

3. Security and Safety

We take security very seriously. The Binary Downloader has several built-in protections:

  • Verification: Every file downloaded is checked against a unique digital "fingerprint" (SHA-256 checksum) stored in our database. If the file doesn't match perfectly, it is immediately deleted and not installed.
  • Isolation: Binaries are kept in a specific, protected folder (hams_bin) where Odoo can run them, but they cannot easily interfere with the rest of your system.
  • Restricted Access: Only administrators can change where binaries are downloaded from or what their fingerprints should be.

4. Troubleshooting

If an installation fails, it's usually for one of these reasons:

  • No Internet Access: The server needs to be able to reach the download URL (usually GitHub).
  • Unsupported System: Currently, automatic installation is specifically optimized for 64-bit Linux systems.
  • Permissions: The Odoo service needs permission to write to its data directory to save the binaries.
