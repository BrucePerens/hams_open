Daemon Key Manager

The Daemon Key Manager is a security utility that handles the "handshake" between Odoo and external background programs (daemons). It eliminates the need to manually create and share passwords by automatically generating secure API keys and placing them in protected files that only the daemon can read.

How it Works

When a module requires an external daemon to talk to Odoo, it registers that daemon with this manager. The manager then:

  1. Generates a secure, long-term API key.
  2. Writes that key, along with the correct username, into a .env file.
  3. Locks the file so that only the Odoo system user can access it.
  4. Automatically replaces the key every 60 days to keep the system secure.

For Site Owners: Managing Daemons

As a site owner or administrator, you can view and manage these connections from the Settings > Security > Daemon Key Registry menu.

Viewing Registered Daemons

The registry list shows you every external program currently authorized to connect to your Odoo instance. You can see:

  • Daemon Name: A friendly name for the external program (e.g., "Cloudflare Purge Worker").
  • Service Account: The specific Odoo user identity the daemon uses. These are restricted "Service Accounts" that cannot log in through the web interface.
  • Environment File Path: Where the secret key is stored on your server's disk.
  • Last Rotated: The last time the key was updated.

Manual Key Refresh

While the system updates keys automatically every 60 days, you might occasionally need to refresh them manually (for example, if you suspect a key might have been compromised). To do this:

  1. Open any daemon record from the registry.
  2. Click the "Force Provision All" button in the top header.
  3. This will immediately invalidate all existing daemon keys and generate fresh ones for all registered daemons.

Security Rules

To keep your server safe, the Daemon Key Manager enforces strict rules:

  • Protected Location: All key files must be stored inside /var/lib/odoo/daemon_keys/. The system will block attempts to write keys to sensitive system folders like /etc/ or /root/.
  • No Admin Access: Keys can only be created for "Service Accounts." You cannot create an API key for a regular administrator or the system superuser using this tool.
  • Automatic Cleanup: When a key is rotated, the old key is immediately deleted from Odoo, ensuring it can never be used again.

What to do if a Daemon stops working?

If an external program suddenly loses its connection to Odoo, it usually means its key has expired or been rotated. Well-designed daemons will automatically detect this, re-read their .env file, and continue working. If a daemon remains disconnected, ensure that it has permission to read the file in /var/lib/odoo/daemon_keys/ and that the Odoo "Cron" jobs are running correctly.
