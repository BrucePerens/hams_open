Daemon Key Manager

The Daemon Key Manager is a security tool that manages the connection between Odoo and external background programs (often called "daemons"). Its job is to make these connections secure and easy to manage without you ever needing to handle passwords manually.

What does this module do?

When an external program needs to talk to Odoo—for example, a service that monitors your server or a tool that automatically clears your website's cache—it needs a way to prove who it is. Usually, people use passwords for this, but passwords can be lost, stolen, or forgotten.

This module replaces passwords with API Keys. An API Key is a long, random string of characters that acts like a single-purpose password. The Daemon Key Manager:

  • Generates these keys automatically.
  • Stores them in a secure file on the server's disk where only the authorized program can find them.
  • Rotates them every 60 days, meaning it throws away the old key and makes a new one, so even if a key were somehow discovered, it would soon become useless.

How to use the Registry

You can find the management screen by going to Settings > Security > Daemon Key Registry.

1. Viewing Connections

The main list shows you all the external programs currently registered to talk to your Odoo system. For each one, you can see:

  • Daemon Name: The name of the external program.
  • Service Account: The "identity" the program uses inside Odoo. These are special accounts that cannot log in through a web browser, making them extra secure.
  • Company: The company associated with this daemon. The module is fully multi-company aware.
  • Environment File Path: The exact location on the server where the key is stored.
  • Last Rotated: When the key was last updated.

2. Refreshing Keys Manually

Normally, the system handles everything automatically. However, if you suspect a key might have been compromised, or if you've just moved your system to a new server, you can refresh the keys immediately.

  1. Open the Daemon Key Registry.
  2. Open the record for the specific program you want to refresh.
  3. Click the "Rotate Key" button. The system will immediately generate a new key for just that program.
  4. If you need to refresh keys for every program at once (for example, after a server migration), click the "Force Provision All" button.
Tip: If an external program stops working after a refresh, it should be designed to "self-heal" by re-reading its configuration file. If it doesn't, you may need to restart that specific external program.

3. Automated Key Rotation

The system is designed to be "set and forget." Every 60 days, a background task automatically:

  • Identifies keys that are about to expire.
  • Generates a fresh replacement key.
  • Updates the secure .env file on the server.
  • Deactivates the old key.

This ensures that even if a key were somehow leaked, it would only be valid for a short window of time.

4. Adding a New Connection Manually

While most external programs register themselves automatically when their corresponding module is installed, you can add one manually:

  1. Click the New button in the Daemon Key Registry.
  2. Daemon Name: Give it a clear, unique name.
  3. Service Account: Choose the specialized Odoo user account this program will use. Note: This user must be marked as a "Service Account" in the Users menu.
  4. Company: Make sure you select the correct Company for this daemon if you are operating in a multi-company environment. Daemons are scoped to specific companies.
  5. Environment File Path: Enter the full path on the server where the key file should be saved. It must start with /opt/hams/etc/keys/.
  6. Click Save. A new key will be generated and written to the file immediately.

Security Rules and Safety

To protect your data, this module follows strict safety rules:

  • Strict Locations: All key files MUST be stored inside the /opt/hams/etc/keys/ folder. The system will block any attempt to save keys in sensitive areas like your system's configuration folders, user home directories, or system logs.
  • No "Superuser" Access: You cannot create a key for the main "System" or "Admin" accounts. Keys are only for restricted "Service Accounts."
  • File Protection: When a key file is created, the system sets the permissions so that only the Odoo process can read or write it.

Common Troubleshooting

A daemon cannot connect

If a daemon says "Access Denied," check the following:

  1. Ensure the Environment File Path in Odoo matches where the daemon is looking for its file.
  2. Ensure the Odoo server has permission to write to that location.
  3. Ensure the Service Account assigned to the daemon is still "Active" in the Users menu.

The "Last Rotated" date is very old

If the keys haven't been updated in more than 60 days, it might mean the Odoo "Scheduled Actions" (Crons) are not running. Ask your system administrator to check if the Odoo Cron runner is active.

Important: Never share the contents of the .env files with anyone. They contain the "master keys" for your background services.
