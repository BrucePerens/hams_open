Zero-Sudo Security Core

The zero_sudo module is the foundational security layer for our entire system. It acts as a "security cop," ensuring that every part of the software only has the permissions it absolutely needs to do its job. It implements two major security rules: Zero-Sudo Architecture and Service Account Isolation.

What is "Zero-Sudo"?

In most software, developers can easily "become the administrator" to perform complex tasks. This is dangerous because if one part of the system is hacked, the attacker can use this "sudo" power to take over everything. Our system forbids this. Instead, we use "Service Accounts" — special accounts that have very limited, specific permissions.

Key Features for Users

1. Service Accounts

As an administrator, you can see which users are Service Accounts by looking for the checkbox on the user form. These accounts are designed for background tasks and external systems (like backup tools or helpdesk bots). To keep things secure, these accounts are physically blocked from logging into the website. If someone tries to use a service account's password in a web browser, the system will immediately kick them out.

2. Safe Configuration

The system maintains a "whitelist" of safe settings. This prevents a common type of hack where an attacker tries to trick the system into revealing sensitive secrets (like database passwords) through the user interface. Only settings that have been explicitly approved by the developers can be accessed through the Zero-Sudo tools.

3. Automatic Documentation

This module also manages all the manuals and guides you are currently reading! It provides a centralized way for all other modules to install their documentation into this library automatically. When a new module is installed, its documentation appears here without any extra setup required by you.

Common Troubleshooting

  • "Access Denied" on Login: If you see an error saying "Access Denied" when trying to log in, and the web address contains access_denied_service, you are trying to log in with a Service Account. Human users must use accounts where "Is Service Account" is unchecked.
  • Missing Settings: If a developer tells you they can't access a specific configuration setting, it's likely because that setting hasn't been added to the security "whitelist" in this module. This is a deliberate safety feature.
