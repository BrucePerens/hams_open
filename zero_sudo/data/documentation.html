Zero-Sudo Security: User Guide

The zero_sudo module is the "security guard" of your system. It works behind the scenes to keep your data safe by making sure that automated tasks and background processes only have the exact permissions they need.

Key Security Features

Service Accounts

A "Service Account" is a special user record created for automated tasks (like a bot that sends emails or a script that backs up your data). Unlike human users, these accounts are restricted to prevent accidents or security breaches.

  • Web Login Block: To keep your system secure, Service Accounts are forbidden from logging in through a web browser. If anyone tries to use a Service Account's credentials at the login screen, they will be instantly blocked and logged out.
  • How to Identify: You can see if an account is a Service Account by looking for the "Is Service Account" checkbox on their user profile in the settings menu.

Safe Configuration

The system maintains a "whitelist" (a list of allowed items) for sensitive settings. This prevents even an authorized process from accidentally accessing or changing critical security information like master passwords or encryption keys unless it is explicitly allowed to do so.

Global Data Management

Some information in the system is considered "Global", meaning it is shared across all websites and companies. This is typical for technical system data.

  • System Keys (KV Store): Technical state and security flags are stored centrally to ensure consistency across the entire platform. [@ANCHOR: zero_sudo_kv_global]
  • Leak Detection (Noisy Tables): The names of database tables that are safe to ignore during security audits are stored globally. [@ANCHOR: zero_sudo_noisy_table_global]

Automatic Help Library

This module also helps organize the very library you are reading! It automatically finds and installs help guides for every new feature you add to your system, ensuring your team always has the latest instructions available in the Knowledge base.

Frequently Asked Questions

"I get an 'Access Denied' error when logging in!"

If you see this error (specifically access_denied_service), it means you are trying to log in with an account that is reserved for background tasks. Please use your standard employee account to access the website.

"Where are my manuals?"

Documentation is automatically updated and installed whenever you add new modules to the system. If a manual is missing, it might be because the module is not yet fully installed or active.

Management for Administrators

Administrators can manage which accounts are restricted by going to Settings > Users & Companies > Users and checking or unchecking the Is Service Account box. Only check this box for accounts that do not need to use the web interface.
