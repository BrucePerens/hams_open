Backup Management User Guide

This module centralizes the management of your system's data protection. It supports both file-level backups (Kopia) and database-level backups (pgBackRest).

Core Features

  • Unified Dashboard: View the health and status of all your backup configurations in one place.
  • Automated Scheduling: Configure periodic snapshots and retention policies.
  • Safe Execution: Operations run in the background to ensure the system remains responsive.
  • Integrity Checks: Automated restore drills verify that your data can actually be recovered.

Getting Started

1. Setup a Backup Configuration

Go to Backup Management > Configuration > Backup Configurations. Create a new record, choosing the appropriate engine (Kopia for files, pgBackRest for databases) and providing the target destination. You can also specify how many daily, weekly, and monthly snapshots to keep.

2. Monitor via Dashboard

The Backup Dashboard shows you at a glance if your backups are up to date. Records highlighted in red indicate that no backup has been successful in the last 26 hours. The dashboard also displays the status of the most recent backup or sync job, providing real-time visibility into background operations.

3. Performing a Restore

In the event of data loss, navigate to Backups > Snapshots. Select a known good snapshot and click the Restore button. You will be prompted for a target directory where the files should be placed.

Security Note: Only users with "Backup Administrator" privileges can create configurations or trigger restore operations.
