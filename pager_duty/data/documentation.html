Overview

The Pager Duty module is your central hub for infrastructure health and Site Reliability Engineering (SRE). It provides proactive monitoring, intelligent alerting, and automated incident management to keep your Odoo instance and surrounding infrastructure running at peak performance. By combining intuitive configuration within Odoo with high-frequency background daemons, it ensures that critical issues are detected and routed to the right person instantly.

Getting Started

1. Configuring Monitoring Checks

To begin monitoring your infrastructure, navigate to Pager Duty > Monitoring Checks. Here, you can define various types of health checks:

  • Web Resources: Monitor HTTP/HTTPS and even modern HTTP/3 (QUIC) endpoints. You can verify not just the status code, but also ensure specific content is present in the response.
  • Database Health: Perform deep health checks on PostgreSQL and MySQL/MariaDB. For PostgreSQL, you can even define custom SQL queries for advanced anomaly detection.
  • System Metrics: Keep a close eye on CPU usage, RAM availability, Disk space, and system Load Averages.
  • Network Services: Verify the availability of TCP/UDP sockets, DNS resolution, Redis, RabbitMQ, Memcached, SSH, and more.
  • Hardware Monitoring: Proactively monitor physical hard drive health using SMART data to predict and prevent failures.
  • Service Management: Automatically track the status of critical systemd services and Docker containers.

2. Synchronizing with Daemons

The monitoring logic is executed by standalone Python programs (daemons) for maximum reliability and performance. After configuring your checks in Odoo, you must sync them:

  1. Go to the Monitoring Checks list view.
  2. Click JSON Configuration Tools and select Export to JSON.
  3. This updates the daemon configuration file (pager_config.json), and the background services will pick up the changes on their next cycle.

3. Managing On-Call Schedules

Alerts are routed based on the Odoo Calendar. To define who is on call:

  • Create a standard Calendar event for the relevant engineer.
  • Check the "Is Pager Duty Shift" box on the event.
  • The system will automatically identify the active shift and route incident notifications (via chatter and email) to that specific user.

Incident Management Lifecycle

Detection and Reporting

When a daemon detects a failure, it immediately reports an Incident to Odoo. To prevent "alert fatigue" during major outages, the system implements automatic Rate Limiting and De-duplication. If multiple failures occur for the same source, they are grouped into a single active incident.

NOC Dashboard

Access the NOC Dashboard via Pager Duty > NOC Dashboard for a real-time, high-visibility overview of your system health.

  • Live Updates: The board refreshes instantly as incidents are created, acknowledged, or resolved.
  • Burn-In Protection: Designed for long-term display on office monitors, the dashboard subtly shifts its layout every 60 seconds to protect your hardware.
  • Mean Time Metrics: View real-time MTTA (Time to Acknowledge) and MTTR (Time to Resolve) analytics.

Acknowledgement & Escalation

  • Acknowledge: When you start working on an issue, click "Acknowledge" on the dashboard or incident record. This signals to the team that the issue is being handled and stops the automatic escalation timer.
  • Escalation: If a high-severity incident remains unacknowledged for more than 15 minutes, the system automatically escalates it by notifying all Pager Duty Administrators.

Recovery and Resolution

In most cases, Pager Duty is "self-healing." Once the underlying issue is fixed and the daemon detects a healthy state, it will automatically transition the incident to Resolved. You can also manually resolve incidents if necessary.

Advanced Technical Features

Multi-Website Awareness

In multi-tenant Odoo deployments, Pager Duty can isolate checks, incidents, and schedules by website. This allows different departments or clients to manage their own independent monitoring and on-call stack. The NOC Dashboard, incident reporting, and on-duty lookups all respect the website_id context.

Zero-Sudo & Micro-Privileges

The monitoring daemons communicate with Odoo using the pager_service_internal service account. This account is configured with minimal privileges necessary to read monitoring checks and report incidents. The system strictly avoids the use of sudo(), ensuring that all operations are performed within the scope of the assigned service user via with_user(). Furthermore, high-privilege operations like binary provisioning are protected by strict allow-lists (see rpc_ensure_executable).

Helpdesk Integration

If the hams_helpdesk module is installed, Pager Duty can be configured to automatically generate formal Helpdesk Tickets for every incident. This provides a structured workflow for post-mortems and long-term issue tracking. The system will even link the current on-call engineer as the ticket assignee.

Log Analyzer (Splunk-style)

The integrated Log Analyzer tails system logs in real-time. You can define regex patterns in Odoo to watch for critical errors (like "filesystem corruption" or "out of memory"). The Interactive Log Search tool allows you to perform live regex searches across your server's log files directly from the Odoo UI.

Synthetic Journeys & Sandboxing

For complex application testing, you can write Playwright (Python) or Bash scripts to simulate user behavior. These scripts are executed in a strictly airgapped and chrooted Bubblewrap (bwrap) sandbox, ensuring maximum security for your monitoring infrastructure.

Troubleshooting & Support

If the NOC Dashboard shows a "Search timeout" or "Daemon Offline" message, ensure that the background services (generalized_monitor.py, etc.) are active on your server. Administrators should check the systemd journal for detailed daemon logs.
