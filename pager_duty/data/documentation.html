Overview

The Pager Duty module is your central hub for infrastructure health. It monitors your servers, services, and hardware, alerting the right people when something goes wrong. It works by combining easy-to-use configuration in Odoo with powerful background programs (daemons) that do the heavy lifting.

How to Use This Module

1. Setting Up Your Checks

To start monitoring, go to Pager Duty > Monitoring Checks. Here, you can create "Checks" for various parts of your system:

  • Websites (HTTP): Checks if your website is up and showing the expected content.
  • Databases (PostgreSQL/MySQL): Ensures your databases are reachable and responsive.
  • Server Resources: Monitors CPU usage, Memory, and Disk space.
  • Services (Systemd/Docker): Confirms that critical background services and containers are running.
  • Hardware (SMART): Predicts hard drive failures before they happen.

Once you've created your checks, use the JSON Configuration Tools button to "Export to JSON". This tells the background programs what to monitor.

2. On-Call Schedules

Pager Duty uses the Odoo Calendar to manage who gets alerted. Create an event in the calendar and check the "Is Pager Duty Shift" box. The system will automatically route alerts to whoever is scheduled for that time.

3. Managing Incidents

When a check fails, an Incident is created. You can view these in the Incidents menu or on the NOC Dashboard.

  • Acknowledge: Click "Acknowledge" to let the team know you are working on it. This stops the automatic escalation.
  • Resolve: Once the issue is fixed, the system will usually resolve the incident automatically. You can also resolve it manually.

4. Automatic Escalation

If an incident isn't acknowledged within 15 minutes, the system will automatically "escalate" it by notifying all Pager Duty Administrators. This ensures that critical problems are never ignored.

5. NOC Dashboard

The NOC (Network Operations Center) Dashboard is designed for a "big screen" display in your office. It shows all active incidents, who is currently on call, and a list of recently resolved issues. It features a subtle "burn-in protection" that shifts the display slightly every minute to protect your monitors.

Advanced Features

  • Maintenance Windows: You can schedule "Maintenance Start" and "End" times on any check to silence alerts while you are doing planned work.
  • Parent Checks: Link checks together. For example, if your main Internet connection is down, the system will silence alerts for individual websites to prevent "alert storms."
  • Log Analysis: The system can "tail" your server logs in real-time and alert you if it sees specific error patterns (like "database corruption").

Troubleshooting

If you see a "Search timeout" when looking at logs, it usually means the background Log Analyzer daemon is not running. Check with your system administrator to ensure all Pager Duty services are active on the server.
