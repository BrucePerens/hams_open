Overview

The Pager Duty module is your central hub for infrastructure health and Site Reliability Engineering (SRE). It provides proactive monitoring, intelligent alerting, and automated incident management.

Getting Started (For Users)

1. Managing On-Call Schedules

Alerts are routed based on the Odoo Calendar. To define who is on call:

  • Create a standard Calendar event for the relevant engineer.
  • Check the "Is Pager Duty Shift" box on the event.
  • The system will automatically identify the active shift and route incident notifications to that specific user.

2. NOC Dashboard

Access the NOC Dashboard via Pager Duty > NOC Dashboard for a real-time, high-visibility overview of your system health.

  • Live Updates: The board refreshes instantly as incidents are created, acknowledged, or resolved.
  • Burn-In Protection: Designed for long-term display on office monitors, the dashboard subtly shifts its layout every 60 seconds to protect your hardware.

3. Acknowledgement & Escalation

  • Acknowledge: When you start working on an issue, click "Acknowledge" on the dashboard or incident record. This signals to the team that the issue is being handled and stops the automatic escalation timer.
  • Escalation: If a high-severity incident remains unacknowledged for more than 15 minutes, the system automatically escalates it by notifying all Pager Duty Administrators.
  • Helpdesk Integration: Use the "Generate Helpdesk Ticket" button on an incident to instantly create a tracked ticket assigned to the on-call responder.

Administrator Guide (For Admins)

1. Configuring Monitoring Checks

To begin monitoring your infrastructure, navigate to Pager Duty > Monitoring Checks. Here, you can define various types of health checks:

  • Web Resources: Monitor HTTP/HTTPS/HTTP3 endpoints.
  • Database Health: Deep health checks on PostgreSQL and MySQL/MariaDB.
  • System Metrics & Network Services: Monitor CPU, RAM, Disk space, and ports.
  • Hardware Monitoring: SMART hard drive monitoring.

2. Autodiscovery

Admins can use the "Autodiscover" button in the Monitoring Checks list view. This action automatically scans your server to detect running web services, databases, and Let's Encrypt certificates, generating recommended checks for you.

3. Manual Triggering

While configuring a check, an Admin can click the "Trigger Check" button to force an immediate execution. This is useful for validating new monitoring rules before committing them to the daemon schedule.

4. Synchronizing with Daemons

After configuring checks in Odoo, you must sync them to the external Python daemons:

  1. Go to the Monitoring Checks list view.
  2. Click JSON Configuration Tools and select Export to JSON (or Import to pull changes).
  3. This updates the daemon configuration file (pager_config.json).

5. Multi-Website Awareness & Security

Pager Duty can isolate checks, incidents, and schedules by website. Data is secured via micro-privileges and a pager_service_internal zero-sudo account.

6. Log Analyzer (Splunk-style)

The integrated Log Analyzer tails system logs in real-time. Define regex patterns to watch for critical errors. The Interactive Log Search tool allows Admins to perform live regex searches across server log files directly from the Odoo UI.

7. Synthetic Journeys & Sandboxing

Admins can write Playwright (Python) or Bash scripts to simulate user behavior. These are executed in a strictly airgapped Bubblewrap (bwrap) sandbox.
